HTML 5 Content Types

Github Page

In HTML 4.01, there was a distinction between block-level and inline elements. In HTML 5, this distinction between block and inline has been replaced with a more complex set of content categories.

Each element in HTML falls into zero or more categories that group elements with similar characteristics.

Content type categories

Metadata content

Metadata content is content that sets up the presentation or behaviour of the rest of the content, or that sets up the relationship of the document with other documents.

Flow content

Most elements that are used in the body of documents and applications are categorised as flow content. The only elements that are not considered flow elements are a subset of metadata elements.

Sectioning content

Sectioning content is content that defines the scope of headings and footers.

Heading content

Heading content defines the header of a section.

Phrasing content

Phrasing content is the text of the document, as well as elements that mark up that text within paragraphs.

Embedded content

Embedded content is content that imports another resource into the document.

Interactive content

Interactive content is content that is specifically intended for user interaction.

Flow exceptions

These elements are considered flow if the following conditions are met:

  • [1] <area>, if it is a descendant of a <map> element
  • [2] <link>, if the itemprop attribute is present
  • [3] <meta>, if the itemprop attribute is present
  • [4] <style>, if the scoped attribute is present
  • [5] <a>, if it contains only phrasing content

Phrasing exceptions

These elements are considered phrasing if the following conditions are met:

  • [6] <area>, if it is a descendant of a <map> element
  • [7] <del>, if it contains only phrasing content
  • [8] <ins>, if it contains only phrasing content
  • [9] <link>, if the itemprop attribute is present
  • [10] <map>, if it contains only phrasing content
  • [11] <meta>, if the itemprop attribute is present

Interactive exceptions

These elements are considered interactive if the following conditions are met:

  • [12] <a>, if the href attribute is present
  • [13] <audio>, if the controls attribute is present
  • [14] <img>, if the usemap attribute is present
  • [15] <input>, if the type attribute is not in the hidden state
  • [16] <menu>, if the type attribute is in the toolbar state
  • [17] <object>, if the usemap attribute is present
  • [18] <video>, if the controls attribute is present

These content categories are defined in more detail in the HTML Living Standard specification.
